## 1. Header / Frontmatter

Setiap file blog MDX harus diawali dengan frontmatter YAML. Berikut adalah field-field yang tersedia:

### Field Wajib

| Field | Tipe | Deskripsi | Contoh |
|-------|------|-----------|--------|
| `title` | string | Judul blog post | `"Tips Belajar React"` |
| `publishedAt` | string (YYYY-MM-DD) | Tanggal publikasi | `"2026-02-02"` |
| `summary` | string | Ringkasan singkat untuk SEO dan preview | `"Panduan lengkap..."` |
| `tag` | string | Kategori blog post | `"Journal"`, `"Tips"`, `"Career & Mindset"` |

### Contoh Frontmatter

```yaml
---
title: "Judul Blog Post Kamu"
publishedAt: "2026-02-02"
summary: "Ringkasan singkat yang muncul di SEO dan preview card."
tag: "Journal"
---
```

### Kategori Tag yang Tersedia

- `Journal` ‚Äî Catatan harian, pengalaman pribadi, curhat developer
- `Tips` ‚Äî Tutorial, tips & trik, panduan praktis
- `Career & Mindset` ‚Äî Karir, motivasi, pengembangan diri
- `Documentation` ‚Äî Dokumentasi teknis (seperti file ini)

## 2. Komponen MDX yang Tersedia

Website ini menyediakan beberapa komponen kustom yang bisa kamu gunakan dalam file MDX.

### 2.1 CodeBlock

Komponen untuk menampilkan kode dengan syntax highlighting dan tombol copy.

#### Props

| Prop | Tipe | Default | Deskripsi |
|------|------|---------|-----------|
| `codeInstances` | array | required | Array objek kode yang akan ditampilkan |
| `className` | string | `""` | Class CSS tambahan (contoh: `"my-10"`, `"my-24"`) |
| `compact` | boolean | `false` | Mode kompak tanpa padding berlebih |
| `copyButton` | boolean | `false` | Menampilkan tombol copy |

#### Struktur codeInstances

```typescript
{
  code: string,      // Kode yang akan ditampilkan
  label: string,     // Label tab (jika multiple)
  language: string   // Bahasa: 'js', 'ts', 'tsx', 'json', 'css', dll
}
```

#### Contoh Penggunaan

<CodeBlock className="my-10"
    compact
    codeInstances={[
        {
            code: 'const greeting = "Hello World";\nconsole.log(greeting);',
            label: 'Javascript',
            language: 'js'
        },
    ]}
    copyButton
/>

#### Contoh Multiple Code Tabs

<CodeBlock className="my-10"
    compact
    codeInstances={[
        {
            code: 'function add(a, b) {\n  return a + b;\n}',
            label: 'JavaScript',
            language: 'js'
        },
        {
            code: 'const add = (a: number, b: number): number => {\n  return a + b;\n}',
            label: 'TypeScript',
            language: 'ts'
        },
        {
            code: 'def add(a, b):\n    return a + b',
            label: 'Python',
            language: 'python'
        },
    ]}
    copyButton
/>

### 2.2 SmartImage (Standar MDX)

Untuk menyisipkan gambar, gunakan syntax markdown standar. Gambar akan otomatis menggunakan komponen SmartImage dengan fitur:

- **Enlarge** ‚Äî Klik gambar untuk memperbesar
- **Aspect Ratio** ‚Äî 16/9 secara default
- **Rounded Corner** ‚Äî Radius medium
- **Margin** ‚Äî Margin otomatis (my-20)

#### Syntax

```markdown
![Alt text](/images/path/to/image.png)
```

#### Contoh

```markdown
![Screenshot Dashboard](/images/projects/project-01/dashboard.png)
```

#### Lokasi Gambar

Simpan gambar di folder `public/images/` dengan struktur:
- `public/images/projects/` ‚Äî Untuk screenshot project
- `public/images/gallery/` ‚Äî Untuk foto galeri
- `public/images/blog/` ‚Äî Untuk ilustrasi blog (buat folder jika perlu)

---

### 2.3 CustomLink

Link otomatis menggunakan komponen SmartLink yang menangani:

- **Internal links** (`/path`) ‚Äî Menggunakan client-side navigation
- **Anchor links** (`#section`) ‚Äî Smooth scroll ke section
- **External links** (`https://...`) ‚Äî Otomatis membuka tab baru dengan `target="_blank" rel="noopener noreferrer"`

#### Syntax

```markdown
[Text Link](https://example.com)
[Link Internal](/about)
[Link ke Section](#heading-id)
```

#### Contoh

```markdown
DM aja gue di [WhatsApp](https://wa.me/628xxxxxxxxxx)
Baca juga [tentang saya](/about)
Lompat ke [bagian tips](#tips-berguna)
```

### 2.4 Heading dengan Anchor Link

Semua heading (H1-H6) otomatis memiliki:

- **Anchor link** ‚Äî ID otomatis berdasarkan teks heading
- **HeadingLink component** ‚Äî Hover untuk melihat link icon
- **Margin** ‚Äî Spacing otomatis

#### Auto-generated ID

ID di-generate dari teks heading dengan format:
- Lowercase
- Spasi diganti dengan hyphen (`-`)
- Karakter non-alphanumeric dihapus
- Multiple hyphens digabung

#### Contoh

```markdown
## Tips Belajar React
```

Akan menghasilkan ID: `#tips-belajar-react`

Kamu bisa link ke heading tersebut dengan:
```markdown
[Lompat ke tips](#tips-belajar-react)
```

## 3. Tag Markdown yang Didukung

Berikut adalah tag markdown standar yang didukung dengan styling otomatis:

### 3.1 Heading

```markdown
# Heading 1 (H1)
## Heading 2 (H2)
### Heading 3 (H3)
#### Heading 4 (H4)
##### Heading 5 (H5)
###### Heading 6 (H6)
```

> **Rekomendasi:** Gunakan H2 untuk section utama, H3 untuk subsection.

---

### 3.2 Bold dan Italic

```markdown
**Teks tebal** atau __teks tebal__
*Teks miring* atau _teks miring_
***Tebal dan miring***
~~Teks dicoret~~ (jika didukung)
```

Hasil:
- **Teks tebal**
- *Teks miring*
- ***Tebal dan miring***

---

### 3.3 Blockquote

```markdown
> Ini adalah quote singkat.

> **Fakta Pahit:** Startup sekelas Facebook dibangun pake PHP.
```

Hasil:
> Ini adalah quote singkat.

> **Fakta Pahit:** Startup sekelas Facebook dibangun pake PHP.

---

### 3.4 List

#### Unordered List (Bullet)

```markdown
- Item pertama
- Item kedua
  - Sub item A
  - Sub item B
- Item ketiga
```

Hasil:
- Item pertama
- Item kedua
  - Sub item A
  - Sub item B
- Item ketiga

#### Ordered List (Numbered)

```markdown
1. Langkah pertama
2. Langkah kedua
   1. Sub langkah A
   2. Sub langkah B
3. Langkah ketiga
```

Hasil:
1. Langkah pertama
2. Langkah kedua
   1. Sub langkah A
   2. Sub langkah B
3. Langkah ketiga

---

### 3.5 Divider (Horizontal Rule)

```markdown
---
```

Hasil: Garis horizontal pemisah antar section

---

### 3.6 Inline Code

```markdown
Gunakan `console.log()` untuk debugging.
Variabel `const` tidak bisa di-reassign.
```

Hasil:
Gunakan `console.log()` untuk debugging.
Variabel `const` tidak bisa di-reassign.

## 5. Tips dan Best Practices

### Struktur File Blog

```
src/app/blog/posts/
‚îú‚îÄ‚îÄ nama-file-blog.mdx          # Gunakan kebab-case
‚îú‚îÄ‚îÄ judul-panjang-blog.mdx      # Semua lowercase, spasi jadi hyphen
‚îî‚îÄ‚îÄ BLOG_WRITING_GUIDE.mdx      # Dokumentasi ini
```

### Naming Convention File

- ‚úÖ `tips-belajar-react.mdx`
- ‚úÖ `kenapa-vue-keren.mdx`
- ‚ùå `Tips Belajar React.mdx` (ada spasi)
- ‚ùå `tips_belajar_react.mdx` (pake underscore)

### Penulisan Judul

- **Capitalize each word** untuk title di frontmatter
- Gunakan **quote** jika ada karakter khusus
- Maksimal 60-70 karakter untuk SEO yang baik

```yaml
# ‚úÖ Bagus
title: "Cara Belajar React untuk Pemula"

# ‚úÖ Dengan karakter khusus
title: "Kenapa 'useEffect' Itu Penting"

# ‚ùå Terlalu panjang
title: "Panduan Lengkap Belajar React dari Nol Sampai Mahir dengan Next.js dan TypeScript"
```

### Penulisan Summary

- 1-2 kalimat saja
- Maksimal 150-160 karakter
- Jelaskan **value** yang didapat pembaca
- Gunakan kata kunci yang relevan

```yaml
# ‚úÖ Bagus
summary: "Pelajari 5 cara debug efektif yang sering dipakai developer senior."

# ‚ùå Terlalu pendek
summary: "Tips debug"

# ‚ùå Terlalu panjang
summary: "Dalam artikel ini kita akan membahas banyak sekali hal tentang debugging dan bagaimana cara melakukannya dengan baik dan benar sesuai best practices..."
```

### Format Tanggal

Selalu gunakan format **YYYY-MM-DD**:

```yaml
# ‚úÖ Benar
publishedAt: "2026-02-02"

# ‚ùå Salah
publishedAt: "02/02/2026"
publishedAt: "2 Februari 2026"
```

### Spacing dan Margin

- Gunakan `className="my-10"` atau `className="my-24"` untuk CodeBlock
- `my-10` = margin 40px (kecil)
- `my-24` = margin 96px (besar)
- Selalu ada baris kosong sebelum dan sesudah heading

### Heading Hierarchy

```markdown
# H1 - Jangan dipakai (sudah ada dari title)

## H2 - Gunakan untuk section utama

### H3 - Gunakan untuk subsection

#### H4 - Gunakan jika perlu detail lebih dalam
```

### Quotes yang Efektif

Gunakan blockquote (`>`) untuk:
- Highlight insight penting
- Kutipan dari orang lain
- Fakta menarik yang ingin ditekankan

```markdown
> **Key Takeaway:** Fokus ke progress, bukan perfection.
```

### Link yang Baik

```markdown
<!-- Internal link -->
Baca juga [pengalaman saya](/blog/hari-pertama-jadi-developer)

<!-- External link (otomatis tab baru) -->
Referensi dari [MDN Web Docs](https://developer.mozilla.org)

<!-- Link ke heading (anchor) -->

```

### Checklist Sebelum Publish

- [ ] Frontmatter lengkap dan valid
- [ ] Tanggal `publishedAt` sudah benar
- [ ] Tag sudah sesuai kategori
- [ ] Semua link berfungsi
- [ ] Gambar (jika ada) sudah di-upload ke `public/images/`
- [ ] Tidak ada typo berarti
- [ ] CodeBlock bisa di-copy
- [ ] Preview di local berjalan dengan baik

## Troubleshooting

### Gambar Tidak Muncul

- Pastikan path benar: `/images/nama-file.png`
- Jangan pakai `./` atau `../`
- File harus ada di folder `public/images/`

### CodeBlock Error

- Pastikan prop `codeInstances` adalah array
- Setiap item harus punya `code`, `label`, dan `language`
- Gunakan backtick dalam string dengan escape atau template literal

### Heading ID Tidak Berfungsi

- Pastikan tidak ada karakter khusus di heading
- ID otomatis dibuat dari teks heading
- Cek di inspect element untuk melihat ID yang dihasilkan
